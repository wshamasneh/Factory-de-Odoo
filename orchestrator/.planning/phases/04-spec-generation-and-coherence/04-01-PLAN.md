  <name>Task 1: Create coherence.test.cjs test scaffold and coherence.cjs library</name>
  <files>tests/coherence.test.cjs, odoo-gsd/bin/lib/coherence.cjs</files>
  <behavior>
    - checkMany2oneTargets: passes when comodel_name exists in spec models or registry models; fails when target missing; passes for BASE_ODOO_MODELS (res.partner, res.users, etc.) even if not in registry; handles Many2many and One2many comodel_name too
    - checkDuplicateModels: passes when spec model names are unique and not in registry; fails when spec model name already exists in registry and module is not the owner
    - checkComputedDepends: passes when depends field references resolve to existing fields on the model; fails when field not found; handles dot-notation paths (first segment validation)
    - checkSecurityGroups: passes when groups in access_rules and record_rules are defined in spec security.groups or registry; fails when group undefined
    - runAllChecks: returns status "pass" only when all 4 checks pass; returns status "fail" if any check fails; includes all 4 check results in checks array
    - BASE_ODOO_MODELS: exported Set with ~20 common base models (res.partner, res.users, res.company, res.currency, product.product, product.template, account.move, account.account, account.journal, mail.thread, mail.activity.mixin, uom.uom, ir.attachment, ir.sequence, ir.cron, hr.employee, hr.department, base, res.config.settings, res.country)
    - Edge cases: empty spec models array, empty registry, spec with no computed fields, spec with no security section
  </behavior>
  <action>
    TDD approach: Write tests FIRST in coherence.test.cjs, then implement coherence.cjs.

    1. Create tests/coherence.test.cjs with test fixtures (sample spec and registry objects) and tests for all 6 exported functions plus edge cases. Use node:test and node:assert. Follow the pattern in tests/registry.test.cjs for structure.

    2. Create odoo-gsd/bin/lib/coherence.cjs implementing:
       - BASE_ODOO_MODELS Set (~20 models)
       - checkMany2oneTargets(spec, registry) -- iterate spec.models array, check each Many2one/Many2many/One2many field's comodel_name against spec models Set + registry models keys + BASE_ODOO_MODELS
       - checkDuplicateModels(spec, registry) -- check spec model names against registry model keys; flag if model exists in registry AND registry model's module differs from spec.module
       - checkComputedDepends(spec, registry) -- for fields with compute + depends, verify first segment of each depends path exists as a field on the model (check both spec and registry)
       - checkSecurityGroups(spec, registry) -- collect all group refs from access_rules and record_rules, verify each exists in spec.security.groups or registry security_groups
       - runAllChecks(spec, registry) -- run all 4, aggregate status
       - cmdCoherenceCheck(cwd, specPath, registryPath, raw) -- load files from disk, call runAllChecks, output result

    CRITICAL: spec.models is an ARRAY (not object). Registry models is an OBJECT keyed by model name. Handle this difference everywhere.

    CRITICAL: Use `const { output, error } = require('./core.cjs');` for CLI output.
