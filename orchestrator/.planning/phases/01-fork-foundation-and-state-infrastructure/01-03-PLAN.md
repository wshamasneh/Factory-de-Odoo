```javascript
function atomicWriteJSON(filePath, data) {
  const backupPath = filePath + '.bak';
  if (fs.existsSync(filePath)) fs.copyFileSync(filePath, backupPath);
  const tmpPath = filePath + '.tmp';
  fs.writeFileSync(tmpPath, JSON.stringify(data, null, 2) + '\n', 'utf-8');
  fs.renameSync(tmpPath, filePath);
}
```

    - Test: Modules with no deps -> tier "foundation" (depth 0)
    - Test: Modules depending on foundation -> tier "core" (depth 1)
    - Test: Modules depending on core -> tier "operations" (depth 2)
    - Test: Modules at depth 3+ -> tier "communication"

    **Generation blocking (DEPG-05):**
    - Test: Module whose deps are all "generated" or beyond -> can generate (returns true)
    - Test: Module with a dep still "planned" -> blocked (returns false with reason)
    - Test: Module with no deps -> always can generate
  </behavior>
  <action>
    **Write tests first (RED), then implement (GREEN).**

    Create `odoo-gsd/bin/lib/dependency-graph.cjs`:

    1. `const { output, error } = require('./core.cjs');`

    Internal functions:

    1. `topoSort(modules)` -- DFS-based topological sort with cycle detection:
       - `modules` is object: `{ name: { depends: [dep1, dep2] } }`
       - Track `visited` (done) and `visiting` (in current DFS path) sets
       - On cycle: throw Error with cycle path in message
       - Returns array of module names in dependency order (deps before dependents)

    2. `computeTiers(modules)` -- Assign tier labels based on max dependency depth:
       - Run topoSort first (ensures no cycles)
       - For each module in sorted order: depth = max(depths of deps) + 1 (or 0 if no deps)
       - Map depth to tier: 0="foundation", 1="core", 2="operations", 3+="communication"
       - Return `{ tiers: { foundation: [...], core: [...] }, depths: { mod: N }, order: [...] }`

    CLI functions:

    3. `cmdDepGraphBuild(cwd, raw)` -- Read module_status.json, extract depends from each module, build adjacency list, return graph structure
    4. `cmdDepGraphOrder(cwd, raw)` -- Read modules, run topoSort, return ordered array
    5. `cmdDepGraphTiers(cwd, raw)` -- Read modules, run computeTiers, return tier structure
    6. `cmdDepGraphCanGenerate(cwd, moduleName, raw)` -- Check if all dependencies of moduleName have status >= "generated" (generated, checked, or shipped). Read module_status.json for current statuses. Return `{ can_generate: bool, blocked_by: [...] }`.

